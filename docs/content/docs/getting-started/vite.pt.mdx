## Crie seu projeto Vite

Comece criando um novo projeto Vite. Este comando criará um novo diretório chamado `my-workflow-app` com uma configuração mínima e configurará um projeto Vite dentro dele.

```bash
npm create vite@latest my-workflow-app -- --template react-ts
```

Entre no diretório recém-criado:

```bash
cd my-workflow-app
```

### Instale `workflow` e `nitro`

```package-install
npm i workflow nitro
```

<Callout>
Embora o Vite forneça as ferramentas de build e o servidor de desenvolvimento, o Nitro adiciona a estrutura de servidor necessária para rotas de API e implantação. Juntos, eles possibilitam construir aplicações full-stack com suporte a fluxos de trabalho. Saiba mais sobre o Nitro [aqui](https://v3.nitro.build).
</Callout>

### Configure o Vite

Adicione `workflow()` à sua configuração do Vite. Isso habilita o uso das diretivas `"use workflow"` e `"use step"`.

```typescript title="vite.config.ts" lineNumbers
import { nitro } from "nitro/vite";
import { defineConfig } from "vite";
import { workflow } from "workflow/vite";

export default defineConfig({
  plugins: [nitro(), workflow()], // [!code highlight]
  nitro: { // [!code highlight]
    serverDir: "./", // [!code highlight]
  }, // [!code highlight]
});
```

## Crie seu primeiro fluxo de trabalho

Crie um novo arquivo para o nosso primeiro fluxo de trabalho:

```typescript title="workflows/user-signup.ts" lineNumbers
import { sleep } from "workflow";

export async function handleUserSignup(email: string) {
  "use workflow"; // [!code highlight]

  const user = await createUser(email);
  await sendWelcomeEmail(user);

  await sleep("5s"); // Pause for 5s - doesn't consume any resources
  await sendOnboardingEmail(user);

  return { userId: user.id, status: "onboarded" };
}

```

Vamos preencher essas funções em seguida, mas vamos dar uma olhada neste código:

* Definimos uma função **fluxo de trabalho** com a diretiva `"use workflow"`. Pense na função do fluxo de trabalho como o _orquestrador_ das **etapas** individuais.
* A função `sleep` do Workflow DevKit nos permite suspender a execução do fluxo de trabalho sem consumir recursos. Um sleep pode durar alguns segundos, horas, dias ou até meses.

## Crie as etapas do seu fluxo de trabalho

Agora vamos definir essas funções que faltam.

```typescript title="workflows/user-signup.ts" lineNumbers
import { FatalError } from "workflow"

// Our workflow function defined earlier

async function createUser(email: string) {
  "use step"; // [!code highlight]

  console.log(`Creating user with email: ${email}`);

  // Full Node.js access - database calls, APIs, etc.
  return { id: crypto.randomUUID(), email };
}

async function sendWelcomeEmail(user: { id: string; email: string; }) {
  "use step"; // [!code highlight]

  console.log(`Sending welcome email to user: ${user.id}`);

  if (Math.random() < 0.3) {
  // By default, steps will be retried for unhandled errors
   throw new Error("Retryable!");
  }
}

async function sendOnboardingEmail(user: { id: string; email: string}) {
  "use step"; // [!code highlight]

  if (!user.email.includes("@")) {
    // To skip retrying, throw a FatalError instead
    throw new FatalError("Invalid Email");
  }

  console.log(`Sending onboarding email to user: ${user.id}`);
}
```

Observando este código:

* A lógica de negócio vive dentro das **etapas**. Quando uma etapa é invocada dentro de um **fluxo de trabalho**, ela é enfileirada para ser executada em uma solicitação separada enquanto o fluxo de trabalho está suspenso, assim como `sleep`.
* Se uma etapa lançar um erro, como em `sendWelcomeEmail`, a etapa será automaticamente tentada novamente até que tenha sucesso (ou atinja a contagem máxima de tentativas da etapa).
* Etapas podem lançar um `FatalError` se o erro for intencional e não deve ser reprocessado.

## Crie seu manipulador de rota

Para invocar seu novo fluxo de trabalho, teremos que adicionar seu fluxo de trabalho a um manipulador de rota de API `POST`, `api/signup.post.ts`, com o seguinte código:

```typescript title="api/signup.post.ts"
import { start } from "workflow/api";
import { defineEventHandler } from "nitro/h3";
import { handleUserSignup } from "../workflows/user-signup";

export default defineEventHandler(async ({ req }) => {
  const { email } = await req.json() as { email: string };
  // Executes asynchronously and doesn't block your app
  await start(handleUserSignup, [email]);
  return {
    message: "User signup workflow started",
  }
});
```

Este manipulador de rota cria um endpoint de requisição `POST` em `/api/signup` que acionará seu fluxo de trabalho.

<Callout>
Fluxos de trabalho podem ser acionados a partir de rotas de API ou de qualquer código do lado do servidor.
</Callout>

## Executar no modo de desenvolvimento

Para iniciar seu servidor de desenvolvimento, execute o seguinte comando no seu terminal no diretório raiz do Vite:

```bash
npm run dev
```

Uma vez que seu servidor de desenvolvimento estiver em execução, você pode acionar seu fluxo de trabalho executando este comando no terminal:

```bash
curl -X POST --json '{"email":"hello@example.com"}' http://localhost:5173/api/signup
```

Verifique os logs do servidor de desenvolvimento do Vite para ver a execução do seu fluxo de trabalho, bem como as etapas que estão sendo processadas.

Além disso, você pode usar a [CLI ou UI Web do Workflow DevKit](/docs/observability) para inspecionar em detalhe as execuções e etapas dos seus fluxos de trabalho.

```bash
# Open the observability Web UI
npx workflow web
# or if you prefer a terminal interface, use the CLI inspect command
npx workflow inspect runs
```
