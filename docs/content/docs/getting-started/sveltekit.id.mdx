## Buat Proyek SvelteKit Anda

Mulailah dengan membuat proyek SvelteKit baru. Perintah ini akan membuat direktori baru bernama `my-workflow-app` dengan setup minimal dan menyiapkan proyek SvelteKit di dalamnya.

```bash
npx sv create my-workflow-app --template=minimal --types=ts --no-add-ons
```

Masuk ke direktori yang baru dibuat:

```bash
cd my-workflow-app
```

### Instal `workflow`

```package-install
npm i workflow
```

### Konfigurasikan Vite

Tambahkan `workflowPlugin()` ke konfigurasi Vite Anda. Ini mengaktifkan penggunaan direktif `"use workflow"` dan `"use step"`.

```typescript title="vite.config.ts" lineNumbers
import { sveltekit } from "@sveltejs/kit/vite";
import { defineConfig } from "vite";
import { workflowPlugin } from "workflow/sveltekit"; // [!code highlight]

export default defineConfig({
  plugins: [sveltekit(), workflowPlugin()], // [!code highlight]
});
```

## Create Your First Workflow

Buat file baru untuk workflow pertama kita:

```typescript title="workflows/user-signup.ts" lineNumbers
import { sleep } from "workflow";

export async function handleUserSignup(email: string) {
  "use workflow"; // [!code highlight]

  const user = await createUser(email);
  await sendWelcomeEmail(user);

  await sleep("5s"); // Pause for 5s - doesn't consume any resources
  await sendOnboardingEmail(user);

  console.log("Workflow is complete! Run 'npx workflow web' to inspect your run")

  return { userId: user.id, status: "onboarded" };
}

```

Kita akan mengisi fungsi-fungsi tersebut nanti, tetapi mari lihat kode ini:

* Kita mendefinisikan fungsi **workflow** dengan direktif `"use workflow"`. Anggap fungsi workflow sebagai _pengatur_ dari **langkah-langkah** individual.
* Fungsi `sleep` dari Workflow DevKit memungkinkan kita untuk menangguhkan eksekusi workflow tanpa menggunakan sumber daya. Durasi sleep bisa beberapa detik, jam, hari, atau bahkan bulan.

## Create Your Workflow Steps

Sekarang mari kita definisikan fungsi-fungsi yang hilang tersebut.

```typescript title="workflows/user-signup.ts" lineNumbers
import { FatalError } from "workflow"

// Our workflow function defined earlier

async function createUser(email: string) {
  "use step"; // [!code highlight]

  console.log(`Creating user with email: ${email}`);

  // Full Node.js access - database calls, APIs, etc.
  return { id: crypto.randomUUID(), email };
}

async function sendWelcomeEmail(user: { id: string; email: string; }) {
  "use step"; // [!code highlight]

  console.log(`Sending welcome email to user: ${user.id}`);

  if (Math.random() < 0.3) {
  // By default, steps will be retried for unhandled errors
   throw new Error("Retryable!");
  }
}

async function sendOnboardingEmail(user: { id: string; email: string}) {
  "use step"; // [!code highlight]

  if (!user.email.includes("@")) {
    // To skip retrying, throw a FatalError instead
    throw new FatalError("Invalid Email");
  }

  console.log(`Sending onboarding email to user: ${user.id}`);
}
```

Perhatikan kode ini:

* Logika bisnis berada di dalam **langkah-langkah**. Ketika sebuah langkah dipanggil di dalam sebuah **workflow**, langkah tersebut akan dimasukkan ke antrean untuk dijalankan pada permintaan terpisah sementara workflow ditangguhkan, sama seperti `sleep`.
* Jika sebuah langkah melemparkan error, seperti pada `sendWelcomeEmail`, langkah tersebut akan dicoba ulang secara otomatis sampai berhasil (atau mencapai batas maksimum percobaan ulang langkah).
* Langkah dapat melempar `FatalError` jika error bersifat disengaja dan tidak boleh dicoba ulang.

## Buat Penangan Rute Anda

Untuk memanggil workflow baru Anda, kita harus menambahkan workflow Anda ke penangan rute API `POST`, `src/routes/api/signup/+server.ts` dengan kode berikut:

```typescript title="src/routes/api/signup/+server.ts"
import { start } from "workflow/api";
import { handleUserSignup } from "../../../../workflows/user-signup";
import { json, type RequestHandler } from "@sveltejs/kit";

export const POST: RequestHandler = async ({
  request,
}: {
  request: Request;
}) => {
  const { email } = await request.json();

  // Executes asynchronously and doesn't block your app
  await start(handleUserSignup, [email]);

  return json({ message: "User signup workflow started" });
};

```

Penangan rute ini membuat endpoint permintaan `POST` di `/api/signup` yang akan memicu workflow Anda.

<Callout>
Workflow dapat dipicu dari rute API atau kode sisi-server apa pun.
</Callout>

## Jalankan di lingkungan pengembangan

Untuk memulai server pengembangan Anda, jalankan perintah berikut di terminal pada direktori root SvelteKit:

```bash
npm run dev
```

Setelah server pengembangan berjalan, Anda dapat memicu workflow Anda dengan menjalankan perintah ini di terminal:

```bash
curl -X POST --json '{"email":"hello@example.com"}' http://localhost:5173/api/signup
```

Periksa log server pengembangan SvelteKit untuk melihat eksekusi workflow Anda serta langkah-langkah yang sedang diproses.

Selain itu, Anda dapat menggunakan [CLI atau UI Web Workflow DevKit](/docs/observability) untuk memeriksa jalannya workflow dan langkah-langkah secara rinci.

```bash
# Open the observability Web UI
npx workflow web
# or if you prefer a terminal interface, use the CLI inspect command
npx workflow inspect runs
```
