---
title: FatalError
---

Lorsqu'un `FatalError` est lancé dans une étape, cela indique que le workflow ne doit pas réessayer l'étape, la marquant comme un échec.

Vous devez l'utiliser lorsque vous ne souhaitez pas qu'une étape spécifique soit réessayée.

```typescript lineNumbers
import { FatalError } from "workflow"

async function fallibleWorkflow() {
    "use workflow"
    await fallibleStep();
}

async function fallibleStep() {
    "use step"
    throw new FatalError("Fallible!") // [!code highlight]
}
```
